You are a senior Python backend architect.
Generate a production-ready Python project that implements a semantic retrieval chatbot over schema.org metadata using local embeddings and FAISS, with an external LLM endpoint used only for reasoning/reranking.

The system must support queries such as:

User Query Example:
"I want 20 items of XYZ stocks"

The system should search through schema.org structured metadata and return the best matching entities.

---

SYSTEM REQUIREMENTS

The system must:

1. Use Python 3.11+
2. Use FastAPI for the API layer
3. Use Sentence Transformers for embeddings
4. Use FAISS as an in-memory vector database
5. Support schema.org JSON-LD input
6. Convert schema data into searchable text
7. Store embeddings in FAISS
8. Perform semantic retrieval
9. Send retrieved candidates to a configured LLM endpoint
10. Return ranked results

No external embedding APIs should be used.

SCHEMA PARSER

Create a module that converts schema.org JSON-LD into text suitable for embeddings.

Example input:

{
"@type": "Product",
"name": "XYZ Semiconductor",
"industry": "Technology",
"exchange": "NASDAQ",
"marketCap": "Large"
}

Converted text:

"XYZ Semiconductor stock in technology industry listed on NASDAQ with large market cap"
